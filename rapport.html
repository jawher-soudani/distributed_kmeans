1. L' algorithme de Kmeans¶

L'algorithme K-means est la mthode de clustering la plus connue et la plus utilise. Elle prend comme paramtre d'entre, k (le nombre de clusters), et partitionne un ensemble de n objets dans k clusters de sorte que la similarit intra-cluster rsultant est lev alors que la similarit inter-cluster est faible.
La similarit entre clusters est mesure en fonction de la valeur moyenne des objets dans le cluster, qui peut tre considr comme le "centre du gravit " du cluster ou mdode.
L'algorithme procde comme suit:

  • Premirement, il slectionne alatoirement k objets parmi tous les objets. Ces points slectionns reprsentent les centres initiaux des clusters.
  • Chacun des points restants est affect au cluster auquel il est le plus similaire, en fonction de la distance entre le point et le centre du cluster. (par exemple, si on utilise la distance euclidienne comme mesure de similarit, on doit affecter le point au cluster dont la distance entre ce point et le centre de cluster est minimale.)
  • Finalement, on met jour le nouveau centrode. Ce processus ritre jusqu' ce que la fonction de perte converge (erreur minimale ou pas de changement des centrodes par rapport l'tape prcdente.).

Pseudo Code de K_means:¶

Supposons qu'on a un ensmeble d'apprentissage: $x^{(1)}, ..., x^{(m)}$, on voulait grouper ces donnes des quelques clusters "cohrents". L'objectif est de prdire k-centrodes et un label $c^{(i)}$ pour chaque point (ce n'est pas de la classification supervise mais $x^{(i)}$ mesure l'appartenance d'un point i l'un des clusters).

1. initialiser les centrodes $\mu_1, \mu_2,..., \mu_k ~ \in \mathbb{R}^n$
2. Rpter jusqu' convergence : {

pour chaque i, $c^{(i)} = \underset{j}{\operatorname{argmin}} ||x^{(i)} - \mu_j ||^2 $

pour chaque j, $\mu_j = \frac{\sum_{i=1}^m \mathbb{1}\{c^{(i)} =j\}x^{(i)}}{\sum_{i=1}^m \mathbb{1}\{c^{(i)} =j\}}$

}

Complexité de k_means:¶

Dans l'algorithme de k-means, le calcul le plus couteux est le calcul des distances. A chaque itration, il faudrait un total de (mk) calcul de distances o m est le nombre de points et k est le nombre de clusters.
Il est vident que les calculs de distances entre un point et les centrodes n'a pas de rapport avec les calculs de distancse entre les autres point avec les centrodes correspondants. Par consquent, les calculs de distances entre diffrents points avec des centrodes peuvent tre excuts en parallle.
Dans chaque itration, les nouveaux centrodes, qui sont utiliss pour de la prochaine itration, devraient tre mis jour. Par consquent les procdures itratives doivent tre excutes en srie.

import numpy as np

class Our_Kmeans():
  
    """cet algorithme de k-means  permet un clustering  pour grouper 
       les points qui ont traits et des prfrences similaires en se 
       basant sur la distance euclidienne et la 2me norme et puis 
       il trace le resultat de clustering avec des points sous forme
       d'etoile qui reprsente les centres trouvs aprs la dernire
       itration
    """
    
    centroids=[]
    data=[]
    
    
    def GetData(self,file):
        """importer les donnes qui sont supposes tre en dimesion 
           2 pouvoir les visualiser , si non l'algorithme fonctionne 
           avec n'importe quelle dimension
        """
        data=pd.read_csv(file, header=None, names=["x1", "x2"], sep=" ")
        return data.values
    
    
    def __init__(self,file):
        """ initier les donnes dans la classe (l'objet)
        """
        self.data=self.GetData(file)
    
    
    def GetCentroids(self,k):
        """ choisir k centre alatoirement pour initier l'algorithme
        """
        initial_centroids = random.sample(list(self.data),k)
        initial_centroids=[lst.tolist() for lst in initial_centroids]
        initial_centroids.sort()
        return initial_centroids
    
    
    def distance_vector(self,v1,v2):
        """cette fonction permet de calculer la distance entre deux point
           quelques soit leur dimension
        """
        diff=[d1 - c2 for d1,c2 in zip(v1,v2)]
        return np.linalg.norm(diff)
    
    
    def diff(self,k,centroids1,centroids2):
        """cette fonction permet de trouver la distance maximale entre
           deux centroides parmi les paires des centroides dans les deux
           vecteur , elle sera utilis pour le test d'arrt de l'algo
           cad si la distance maximale ( qui est le pire cas) est inf-
           rieur  un seuil , on arrte les itrations
        """
        max_distance = 0.0
        for i in range(k):
            dist = self.distance_vector(centroids1[i],centroids2[i])
            if dist > max_distance:
                max_distance = dist

        return max_distance
        
        
    def show(self,X, C, centroids, keep = False):
        """cette fonction permet des visualiser le rsultat finale ainsi
        que les centres trouvs
        """
        plt.figure(figsize=(9,8))
        plt.scatter(X[:, 0], X[:, 1], c=C)        
        plt.plot(centroids[:,0],centroids[:,1],'*m',markersize=20)
        
        
    def kMeans(self,K, plot_progress):
        """ cette fonction est le core de notre algorithme, c'est elle qui
        fait le calcul des centroides et les points les plus proches de 
        chaque centroide pour former les groupes
        """
        self.centroids = self.GetCentroids(K)
        while True:
            # Etape d'affectation de cluster
            C = np.array([np.argmin([np.dot(x_i-y_k, x_i-y_k) for y_k in \
                                     self.centroids]) for x_i in X])
            # Dplacer les centrodes
            newcentroids = [X[C == k].mean(axis = 0) for k in range(K)]
            if self.diff(K,self.centroids,newcentroids)<0.1:
                break
            else:
                self.centroids=newcentroids
            # Tracer les points et les centres
        if plot_progress==1 : self.show(self.data, C, np.array\
                                        (self.centroids))
        return np.array(self.centroids) , C

2.Le kmeans distribué¶

Comme dcrit dans la partie prcdente , l'algorithme de KMeans distribu ncessite un job MapReduce. La fonction map excute la procdure d'assignation de chaque chantillon au plus proche centrode, tandis que la fonction reduce effectue la procdure de mise jour des nouveaux centrodes.
Afin de rduire le cot de transfert via le rseau, une fonction combiner est developpe pour traiter le rsultat des valeurs intermdiaires avec la mme cl pour chaque mapper.

La fonction Map

Pour la fonction Map, l'ensemble des donnes d'entre est stock en tant que fichier de squence des paires <key, value>, chacune reprsentant un point dans l'ensemble des donnes. la cl est l'offset en octets de cette entre par rapport au point de dpart du fichier, et la valeur est les coordonnes de l'entre.
L'ensemble des donnes est divis et globalement diffus tous les mappers. Par consquent, les calculs de distance peuvent s'excuter d'une faon parallle.
Pour chaque mapper, le kmeans distribu utilise au dbut des centrodes qui sont fixs alatoirement sauvegards comme des variables globales. Compte tenu de ces informations, le mapper peut calculer le centrode le plus proche pour chaque point. Ainsi les valeurs intermdiaires sont composes de deux parties: le numro du cluster( qui minimise la distance entre le centrode et le point considr) et les coordonnes du point.

La fonction Combiner

Aprs chaque tche de map, nous appliquons la fonction combiner pour combiner les donnes intermdaires issues d'un mme mapper. Puisque ces donnes intermdiares sont stockes sur le disque local de l'hte, le cot de transfert sur rseau est nul.
Dans la fonction combine, nous additionons partiellement les valeurs assigns au mme cluster. (Remarque:un compbiner peut tre utilis ici car la fonction somme est commutative et associative qui n'est pas le cas pour la fonction moyenne). Afin de calculer la moyenne des points dans le mme cluster, nous devons garder trace du nombre d'chantillons dans le mme cluster pour chaque mapper.

La fonction Reduce

L'entre de la fonction reducer est les donnes obtenues partir la fonction combine (aprs shuffling and sorting). Comme dcrit ci-dessus, les donnes du combiner comprennent une somme partielle des points dans le mme cluster et le nombre d'chantillons de ce cluster.
Dans la fonction de rduction, nous pouvons additionner tous les points et calculer le nombre total de points pour chaque cluster. Par consquent, on peut obtenir les nouveaux centrodes qui vont tre utiliss dans la prochaine itration.

import sys

from mrjob.job import MRJob
import math



class Map_Red_Kmeans(MRJob):
  
    """ la classe mapreduce_kmeans calcule les centrodes  utiliser pour 
        la dernire itration de la carte en utilisant mapreduce mrjob
        qui peut tre excut localement ou sur hadoop ou sur emr
    """

    SORT_VALUES = True
    OUTPUT_PROTOCOL = mrjob.protocol.RawProtocol
    
    
    def distance_vec(self,d,c):
      
        """calcule la distance entre deux vecteurs (en n'importe quelles
           dimensions) 
           np.linalg.norm (point de donnes - centrode)
        """
        
        diff=[d1 - c2 for d1,c2 in zip(d,c)]
        return np.linalg.norm(diff)
 
    def configure_args(self):
        super(Map_Red_Kmeans, self).configure_args()
      
        #les deux mthodes add_passthru_arg et add_file_arg permet 
        #d'ajouter des options autre que le fichier d'entre et de  
        #sortie dans les lignes de commandes
        
        #ajouter le nombre de cluster aprs l'argument --k
        self.add_passthru_arg("--k",  default=4, help="Number of clusters.")
        #ajoute le fichier qui contient les centroides aprs l'argument --c
        self.add_file_arg('--c')

    def file_to_centroids(self):
      
    #rcupre les centoids du fichier (self.option.c) 
    #et les transforme en liste
    
        f = open(self.options.c,'r')
        output_data=f.read().split('\n')
        centroids=[]
        
        #elimier les caracters spciaux et retourner une liste contenant
        #les centroides
        #on passer par cette tape car le rsultat final des centroides
        #retourner par cet algorithme dans le fichier texte est sous cette
        #forme [,,] [,,] ...
        
        for line in output_data:
            if line:
                newline = re.search("\[(.*?)\]", line)
                newline = newline.group()
                newline = newline.replace("[", "").replace("]", "")
                newline.strip()
                k,x,y = newline.split(',')
                centroids.append([float(k),float(x),float(y)])
        f.close()
        return centroids
 
    def mapper(self, _, lines):
        
        """ la fonction map , cette fonction calcule la distance entre
            chaque points et les centroides et atrribue un classe  chaque
            point en se basant sur la distance
            les centroides utiliser au debut sont gnrer alatoirement mais
            aprs, l'algorithme va retourner des centroides et les 
            rutiliser chaque fois pour les amliorer
        """
       
        centroids = self.file_to_centroids()
        for l in lines.split('\n'):
            x,y = l.split(' ')
            p = [float(x),float(y)]
            min_dist=math.inf
            classe = 0
            
            # nous itrons  traver les centrodes, nous avons k centrodes 
            #(nous obtenons le nombre de centrodes de l'argument k en ligne
            #de commande)
            #on calcule la distance entre les centroides et les points
            for i in range(int(self.options.k)):
                dist = self.distance_vec(p,centroids[i][1:])

            #on cherche la distance minimale chaque fois qu'on veut attribu
            #un point  un cluster (centroide)
            
                if dist < min_dist:
                    min_dist = dist
                    classe = centroids[i][0]
            
        yield classe, p
 
    def combiner(self,k,data_points):
    
        """calculer une somme partielle des points suivi d'un coefficient
           avant de calculer la moyenne finale vu que nous ne pouvons pas 
           calculer la moyenne ici ( la moyenne des moyennes de la somme 
           est diffrente de la moyenne de la somme )
           on peux procder par attribuer des coefficients au moyennes
           partielles mais cela ne changera pas la complexit de l'algorithme
           par rapport au calcul de la somme partielle
           les deux methodes permettent de minimiser les flux entre le phase
           de mapping et reducing
        """
      
        first_sum_points = next(data_points)
        first_counter = 1
        for data_point in data_points:
            first_sum_points=[sum(x) for x in zip(first_sum_points\
                                                  ,data_point)]
            first_counter += 1
        yield k, (first_sum_points, first_counter)
    
    
    def reducer(self, cluster, v):   
        """ la fonction reducer permet de calculer la moyenne totale des
            points qui appartiennent au mme custer
            ces moyennes reprsentent les nouveaux centres qui seront
            enregister dans le fichier des centroides et utiliser pour 
            l'itration suivante jusqu'au trouver les centres finals
            
            paramtres : 
            cluster :  classe 
            v :  premire somme partielle des points lis  la classe 
            suivi de son compteur (la somme retourn par le combiner)
        """

        final_sum,final_counter=next(v)
        for partial_sum, first_counter in v:
            final_sum =[sum(x) for x in zip(final_sum,partial_sum)]
            final_counter += first_counter
        yield str(cluster) ,str([x / final_counter for x in final_sum])

Commentaires¶

  • Le language utilis est python pour que notre algorithme pourrait tre comprhensible par tous les data scientist malgr qu'avec java, on aurait pu gagner plus de temps d'execution. Le temps d'exucution de kmeans distribu est beaucoup plus long que celui du kmeans non distribu dans notre cas cause de :

    • petit taille du fichier gnr
    • cluster sur une seule machine
    • cluster locale

  • En utilisant Amazon Elastic Mapreduce (service payant) le rsultat serait incomparable avec un fichier de grosse taille

  • To calculate accuracy for the mapreduce , we need to sort the predicted points and sort the original points so that we can compare their classe because reducers does not save the order in parallel compute (accuracy of our algorithm is about 95,23% a little bit less than the single run kmeans because we are using a small file which is not the case for big files)

  • pour excuter le programme sur une autre machine, il faut faut changer les chemins locaux dans multirun.py

  • pour excuter le programme sur hadoop , il suffit d'ajouter hadoop la ligne de commande d'appel du programme et il faut changer les configuration des arguments dans le main program et il faut prciser hadoop quels sonts les mappers et les reducers, c'est simple vu que mrjob supporte hadoop facilement

  • pour excuter sur amazon elastic map reduce il faut ajouter emr la ligne de commande d'appel du programe

  • si on fait pas appel hadoop ou emr comme dans notre cas , mrjob cre son propre cluster distribu dans la machine locale , cette mthode est gnralement utilis pour le test de l'algorithme avant de l'utiliser avec des gros fichiers dans des plateforme payante qui ne tolre pas le fail d'excution (on paye toujours , c'est le cas de amazon emr)
